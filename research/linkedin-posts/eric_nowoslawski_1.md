### 📝 Core Post Content
> "Here's a new process we've been doing for email deliverability... In the past, we have basically set up 2 batches of inboxes for customers. One that we expected to use, and another that we would use in case things went wrong. 
> 
> We are switching formats so that we are diversified AND we can quite possibly age domains for 3 months before using them. Now we are setting up 4 batches of inboxes for customers. We first estimate how many emails they'll need to be sending per day so that each batch is capable of hitting it. 
> 
> * **Batch 1:** Google inboxes.
> * **Batch 2:** Outlook inboxes provided by our friends at Hypertide.io.
> * **Batch 3:** A combination of both inboxes.
> * **Batch 4:** Another combination of both inboxes.
> 
> We are finding right now to get the best performance with google and outlook inboxes need to warm longer which is why we get started like this. But what we are doing is not using all inboxes at the same time... As domain age becomes more and more important we can lean into this strategy to have warmer and warmer inboxes so that if we ever burn some we can regroup, restrategize, and refresh copy. and launch from new inboxes that are even warmer than the ones we used before."

---

### 💡 Key Takeaways for B2B SaaS
* **Horizontal Scaling & Risk Diversification:** Moving from massive single-source blasts to a structured 4-batch infrastructure (hybrid Google + Outlook) isolates delivery failures and protects domain reputation.
* **Prolonged Domain Warming Strategy:** Keeping reserve inbox batches in an untouched warming pool for up to 3 months provides immediate, highly-reputable backup infrastructure if active domains get flagged.
